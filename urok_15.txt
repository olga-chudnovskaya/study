Создание репозитория и связывание его с репозиторием на GitHub
 
1) создаём папку и заходим в Git Bash кликом правой кнопки мыши (в ФАРе находясь в папке в консоли пишем команды) 

2) создаём репозиторий: git init

3) создаём репозиторий на github.com с тем же именем

4) Создаем файлы в папке и добавляем эти файлы git add .

5) git commit -a -m "комментарий"

6) проверяем git status

7) связываем свой репозиторий с репозиторием на GitHub:
git remote add origin <https://# >

8) git push origin master

или командой git push --set-upstream origin master https://github.com/L-graf/ПАПКА.git создаём ветку Мастер, которая будет связана с нашей веткой Мастер



Создание новой ветки и переход к ней


1) Создаем ветку: git checkout -b <имя ветки> или 
git checkout -b <имя ветки> является шорткатом для git branch <имяветки> за которым идет git checkout <имяветки>

2) Создаем файлы в папке и добавляем эти файлы: git add .

3) git commit -a -m "комментарий"

4) проверяем git status

5) git push <имя новой ветки>

6) git status

Порядок ежедневной работы

1 Проверка обновлений
git pull
2 Проверка состояния
git status
3 Создание новой ветки
git checkout master
git checkout -b <НОВАЯ ВЕТКА>
если нужная ветка уже есть, команда
git checkout <ИМЯ ВЕТКИ>
4 Работа в новой ветке...
5 Сохранение работы
git add .
git commit -a -m "КОММЕНТАРИЙ"
6 Отправка изменений на гитхаб
git push --set-upstream origin <ИМЯ ВЕТКИ>
если в эту ветку уже пушили, команда
git push
