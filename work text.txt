
Создание репозитория и связывание его с репозиторием на GitHub
Порядок ежедневной работы

1 Проверка обновлений
git pull
2 Проверка состояния
git status
3 Создание новой ветки
git checkout master
git checkout -b <НОВАЯ ВЕТКА>
если нужная ветка уже есть, команда
git checkout <ИМЯ ВЕТКИ>
4 Работа в новой ветке...
5 Сохранение работы
git add .
git commit -a -m "КОММЕНТАРИЙ"
6 Отправка изменений на гитхаб
git push --set-upstream origin <ИМЯ ВЕТКИ>
если в эту ветку уже пушили, команда
git push

Страницы сайта.

1)Обычная страница ограниченной высоты.
Состоит из :
 а)Шапки(header)
В шапке располагается лого, поисковая строка или маленький поиск, верхнее меню, может быь кнопка перехода на корзину? входа в личный кабинет.
 

 б)Основное содержимое, состоит чаще всего из 2 - 3 колонок.
В левой колонке распологается к.-л. меню, каталог и т.д.
Основная - контентная часть (основное содержимое)

в)Подвала (footer)
 В нем содержится инфа о копирайте, менб, ссылки на страницы (фоорму обратной связи, контакты, договор офферты, политика конфиденциальности, крупно написаный телефон)

2)Лендинг.
 а)Шапка(header) - лого, контактная инфа, меню перехода по якорям для перехода к секциям.
 б)Секции - одна или несколько
 в)Подвал - может и не быть 
 
Обверстка - вид страницы (повторяющиеся элемены, вид). Может быть одинаковая для всех страниц, а может отличаться для Главной.

Контент - разный на каждой странице.

Хлебные крошки (breadcrumbs) 

Вкладки. На сайтах вкладки применяются там, где требуется компактно и наглядно скомпоновать информацию по блокам. Переход по вкладкам происходит без перезагрузки страницы, что ускоряет работу и упрощает пользователю жизнь.
Код вкладок состоит из двух частей — это отображение самих вкладок, а также информации с ними связанных.
https://webref.ru/layout/bootstrap/component/tabs

	Эскиз — это уменьшенное изображение, одновременно являющееся ссылкой на страницу, либо на то же изображение увеличенное в размерах. Часто используют синоним «ноготки» (от англ. thumbnail — ноготь) или кальку «тамбы».
В отличие от обычных изображений эскизы вписываются в модульную сетку и размеры картинок подстраиваются под ширину макета. Поэтому вначале мы должны сделать сетку, а уже в колонки вставить ссылки с фотографиями. 
https://webref.ru/layout/bootstrap/component/thumb

	Сообщения
Чтобы дать пользователю знать о результатах его действий применяются информационные сообщения — для разных целей используются разные цвета. 
https://webref.ru/layout/bootstrap/component/alert

	Всплывающая подсказка - может быть добавлена к любому элементу HTML с помощью универсального атрибута title. Тогда при наведении курсора на элемент появится текст, заданный внутри title. Оформление такого текста возложено на браузер и никак не меняется через стили. Поэтому часто применяют пользовательские подсказки сделанные через JavaScript. Их можно гибко настраивать по своему вкусу, добавляя возможности, которых лишена встроенная подсказка.
https://webref.ru/layout/bootstrap/component/tooltip
